# MTRODCL - Meter Table DCLGEN

## Overview
MTRODCL is a DB2 DCLGEN (Declaration Generator) member that provides COBOL data structure declarations for the POMS_METER table. This file is automatically generated by DB2 and contains both SQL DECLARE statements and corresponding COBOL record definitions for meter data management.

## Table Definition

### SQL Table Structure
```sql
EXEC SQL DECLARE POMS_METER TABLE
( CUST_ID                        VARCHAR(12) NOT NULL,
  MTR_ID                         VARCHAR(16) NOT NULL,
  PREV_READ                      VARCHAR(6),
  CURR_READ                      VARCHAR(6)
) END-EXEC.
```

### Table Columns
| Column Name | Data Type | Length | Nullability | Description |
|-------------|-----------|--------|-------------|-------------|
| CUST_ID | VARCHAR | 12 | NOT NULL | Customer identifier (foreign key) |
| MTR_ID | VARCHAR | 16 | NOT NULL | Unique meter identifier (primary key) |
| PREV_READ | VARCHAR | 6 | Nullable | Previous meter reading |
| CURR_READ | VARCHAR | 6 | Nullable | Current meter reading |

## COBOL Data Structure

### Meter Record Definition
```cobol
01  METER-RECORD.
    10 MT-CUST-ID.
       49 MT-CUST-ID-LEN    PIC S9(4) USAGE COMP.
       49 MT-CUST-ID-TEXT   PIC X(12).
    10 MT-MTR-ID.
       49 MT-MTR-ID-LEN     PIC S9(4) USAGE COMP.
       49 MT-MTR-ID-TEXT    PIC X(16).
    10 MT-PREV-READ.
       49 MT-PREV-READ-LEN  PIC S9(4) USAGE COMP.
       49 MT-PREV-READ-TEXT PIC X(6).
    10 MT-CURR-READ.
       49 MT-CURR-READ-LEN  PIC S9(4) USAGE COMP.
       49 MT-CURR-READ-TEXT PIC X(6).
```

### Field Descriptions

#### Customer ID (VARCHAR)
- **Length Field**: `MT-CUST-ID-LEN`
- **Text Field**: `MT-CUST-ID-TEXT`
- **Max Length**: 12 characters
- **Purpose**: Foreign key linking to customer table
- **Constraint**: NOT NULL (required field)

#### Meter ID (VARCHAR)
- **Length Field**: `MT-MTR-ID-LEN`
- **Text Field**: `MT-MTR-ID-TEXT`
- **Max Length**: 16 characters
- **Purpose**: Unique meter identification
- **Constraint**: NOT NULL (primary key)

#### Previous Reading (VARCHAR)
- **Length Field**: `MT-PREV-READ-LEN`
- **Text Field**: `MT-PREV-READ-TEXT`
- **Max Length**: 6 characters
- **Purpose**: Previous meter reading value
- **Constraint**: Nullable (can be NULL for new meters)

#### Current Reading (VARCHAR)
- **Length Field**: `MT-CURR-READ-LEN`
- **Text Field**: `MT-CURR-READ-TEXT`
- **Max Length**: 6 characters
- **Purpose**: Current meter reading value
- **Constraint**: Nullable (can be NULL for new meters)

## VARCHAR Handling

### Length Fields
Each VARCHAR field has an associated length field:
- **Format**: `PIC S9(4) USAGE COMP` (binary format)
- **Purpose**: Stores actual length of text data
- **Management**: Automatically handled by DB2 precompiler

### Text Fields
- **Format**: `PIC X(n)` where n is maximum length
- **Purpose**: Contains actual character data
- **Initialization**: Should be initialized to spaces

### Example Usage
```cobol

MOVE 'CUST00000001' TO MT-CUST-ID-TEXT
MOVE 16 TO MT-MTR-ID-LEN
```

## Sample Operations

### Insert New Meter
```cobol
MOVE 12 TO MT-CUST-ID-LEN
MOVE 'CUST00000001' TO MT-CUST-ID-TEXT
MOVE 16 TO MT-MTR-ID-LEN
MOVE 'MTR-2024-C001' TO MT-MTR-ID-TEXT
MOVE 0 TO MT-PREV-READ-LEN
MOVE SPACES TO MT-PREV-READ-TEXT
MOVE 0 TO MT-CURR-READ-LEN
MOVE SPACES TO MT-CURR-READ-TEXT

EXEC SQL
    INSERT INTO POMS_METER
    VALUES (:MT-CUST-ID, :MT-MTR-ID, :MT-PREV-READ, :MT-CURR-READ)
END-EXEC.
```

### Update Meter Reading
```cobol
MOVE 6 TO MT-CURR-READ-LEN
MOVE '001250' TO MT-CURR-READ-TEXT

EXEC SQL
    UPDATE POMS_METER
    SET PREV_READ = CURR_READ,
        CURR_READ = :MT-CURR-READ
    WHERE MTR_ID = :MT-MTR-ID
END-EXEC.
```

### Query Meter Data
