# CUSTODCL - Customer Table DCLGEN

## Overview
CUSTODCL is a DB2 DCLGEN (Declaration Generator) member that provides COBOL data structure declarations for the POMS_CUSTO table. This file is automatically generated by DB2 and contains both SQL DECLARE statements and corresponding COBOL record definitions.

## Table Definition

### SQL Table Structure
```sql
EXEC SQL DECLARE POMS_CUSTO TABLE
( CUSTOMER_ID                    CHAR(12) NOT NULL,
  FIRST_NAME                     VARCHAR(10),
  LAST_NAME                      VARCHAR(10),
  AREA_CODE                      VARCHAR(6),
  ADDRESS                        VARCHAR(30),
  CITY                           VARCHAR(10),
  UNITS                          VARCHAR(5)
) END-EXEC.
```

### Table Columns
| Column Name | Data Type | Length | Nullability | Description |
|-------------|-----------|--------|-------------|-------------|
| CUSTOMER_ID | CHAR | 12 | NOT NULL | Unique customer identifier |
| FIRST_NAME | VARCHAR | 10 | Nullable | Customer's first name |
| LAST_NAME | VARCHAR | 10 | Nullable | Customer's last name |
| AREA_CODE | VARCHAR | 6 | Nullable | Area code for location |
| ADDRESS | VARCHAR | 30 | Nullable | Street address |
| CITY | VARCHAR | 10 | Nullable | City name |
| UNITS | VARCHAR | 5 | Nullable | Utility units |

## VARCHAR Handling

### Length Fields
Each VARCHAR field has an associated length field:
- **Format**: `PIC S9(4) USAGE COMP` (binary format)
- **Purpose**: Stores actual length of text data
- **Management**: Automatically handled by DB2 precompiler

### Text Fields
- **Format**: `PIC X(n)` where n is maximum length
- **Purpose**: Contains actual character data
- **Initialization**: Should be initialized to spaces

### Example Usage
```cobol
MOVE 10 TO CT-FIRST-NAME-LEN
MOVE 'JOHN' TO CT-FIRST-NAME-TEXT
```
