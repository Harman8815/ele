# MTR002 - Meter Batch Processing Program

## Overview
MTR002 is a COBOL batch program that processes meter data from sequential files and customer master data from VSAM KSDS files, then inserts meter records into the POMS_METER table in DB2. This program generates unique meter IDs, validates input data, and manages database transactions for meter information management.

## Purpose
- **Meter Data Processing**: Process meter readings from input files
- **Customer Integration**: Link meter data to customer master records
- **Database Operations**: Insert meter records into DB2 database
- **ID Generation**: Generate unique meter identifiers
- **Error Handling**: Validate data and handle processing errors
- **Reporting**: Provide processing statistics and error reports


### Processing Phase
```cobol
       2000-PROCESS  SECTION.
           PERFORM 2100-OPEN-FILES.
           PERFORM 2200-READ-CUSTOMER-KSDS UNTIL CUST-KSDS-EOF.
```

### File Operations
```cobol
       2100-OPEN-FILES  SECTION.
           OPEN INPUT MI01-CUSTOMER-KSDS.
           OPEN INPUT TI01-METER-FILE.
           OPEN OUTPUT TO01-METER-ERR.
```

### Record Processing
```cobol
       2200-READ-CUSTOMER-KSDS  SECTION.
           READ MI01-CUSTOMER-KSDS
                AT END  SET CUST-KSDS-EOF TO TRUE
                NOT AT END  ADD 1  TO WS-READ-CTR
                            PERFORM 2205-READ-METER-DATA
           END-READ.

       2205-READ-METER-DATA SECTION.
           READ TI01-METER-FILE
                AT END  SET METER-EOF TO TRUE
                NOT AT END  PERFORM 2300-VALIDATE-METER
           END-READ.
```

## Data Validation

### Validation Rules
```cobol
       2300-VALIDATE-METER SECTION.
           SET VALID-RECORD-FLAG TO TRUE.

           IF IN-PREV-READ IS EQUAL TO SPACES
              DISPLAY 'METER PREV READ ERROR - PREV_READ REQUIRED'
              SET ERROR-RECORD-FLAG TO TRUE
              MOVE TI01-METER-RECORD TO TO01-METER-ERR-RECORD
              WRITE TO01-METER-ERR-RECORD
              ADD 1 TO WS-ERROR-CTR
           END-IF.

           IF IN-CURR-READ IS EQUAL TO SPACES
              DISPLAY 'METER CURR READ ERROR - CURR_READ REQUIRED'
              SET ERROR-RECORD-FLAG TO TRUE
              MOVE TI01-METER-RECORD TO TO01-METER-ERR-RECORD
              WRITE TO01-METER-ERR-RECORD
              ADD 1 TO WS-ERROR-CTR
           END-IF.
```

### Validation Criteria
- **Required Fields**: Previous reading and current reading cannot be spaces
- **Data Format**: All fields must contain valid numeric data
- **Record Length**: Input record must be 12 characters
- **Error Handling**: Invalid records written to error file

## Meter ID Generation

### ID Generation Algorithm
```cobol
       2400-INS-METER-DB2 SECTION.
           MOVE SPACES TO MT-MTR-ID.
      *     GENERATE METER ID: MTR-+CUST_INITIALS(2)+DDMM+RANDOM(4)
           MOVE CUST-KEY(2:1)          TO WS-MTR-CUST-CH1.
           MOVE CUST-KEY(3:1)          TO WS-MTR-CUST-CH2.
           MOVE CUST-KEY(6:3)         TO WS-MTR-AREACODE.
           COMPUTE WS-MTR-RAND = FUNCTION RANDOM * 10000.
           STRING WS-MTR-PREFIX WS-MTR-CUST-CH1 WS-MTR-CUST-CH2
                  WS-MTR-AREACODE WS-MTR-RAND
                  DELIMITED BY SIZE
                  INTO MT-MTR-ID
           END-STRING.
```

### ID Format
- **Prefix**: 'MTR' (Meter)
- **Customer Chars**: 2 characters from customer ID
- **Area Code**: 3 characters from customer ID
- **Random**: 4-digit random number
- **Example**: MTRAB1234567

## Error Handling

### SQL Error Processing
```cobol
           MOVE SQLCODE                TO WS-SQL-CODE.
           EVALUATE TRUE
               WHEN SQL-IO-STATUS
                    DISPLAY ' -------------------------------------'
                    DISPLAY 'METER INSERTED', ' ', MT-MTR-ID
                    DISPLAY ' -------------------------------------'
                    ADD 1  TO WS-WRITE-CTR
               WHEN SQL-DUP-ROW
                    DISPLAY ' -------------------------------------'
                    DISPLAY 'METER EXISTS  ', ' ', MT-MTR-ID
                    DISPLAY ' -------------------------------------'
                    ADD 1  TO WS-DUP-CTR
               WHEN OTHER
                    DISPLAY ' -------------------------------------'
                    DISPLAY 'ERROR INSERTING METER',  ':', WS-SQL-CODE
                    DISPLAY 'METER ID: ', MT-MTR-ID
                    DISPLAY ' -------------------------------------'
                    CALL 'DSNTIAR' USING SQLCA
                    ADD 1 TO WS-ERROR-CTR
           END-EVALUATE.
```

### Error Categories
- **SQLCODE = 000**: Successful insert
- **SQLCODE = -803**: Duplicate meter ID
- **SQLCODE < 0**: Other database errors
- **File Errors**: Input/output file issues
