# MID001.cobol - Meter Generation Program Documentation

## Overview
MID001 is a COBOL batch program that processes meter reading data, generates unique meter IDs, validates readings, and creates meter records in a VSAM KSDS file while handling errors appropriately.

## Program Structure

### IDENTIFICATION DIVISION (Lines 1-2)
```cobol
IDENTIFICATION DIVISION.
PROGRAM-ID.  MTR001.
```
- **Purpose**: Meter generation and processing
- **Key Point**: Program ID is MTR001 (not MID001)
- **Viva Tip**: "PROGRAM-ID should match the actual program name used in JCL and system documentation"

### ENVIRONMENT DIVISION (Lines 4-29)
```cobol
ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
```

#### File Definitions (Lines 9-29)
**Four files are defined:**

1. **TI01-METER-FILE** (Lines 9-12)
   - Type: Sequential input file
   - Assignment: METERIN
   - Purpose: Raw meter reading data input
   - **Key Insight**: Simple sequential file with just meter readings

2. **MI01-CUSTOMER-KSDS** (Lines 14-18)
   - Type: Indexed VSAM KSDS
   - Assignment: CUSTKSDS
   - Access: Sequential
   - Record Key: CUST-KEY
   - Purpose: Customer master data for ID generation
   - **Key Insight**: Sequential access processes all customers

3. **MO01-METER-KSDS** (Lines 20-24)
   - Type: Indexed VSAM KSDS
   - Assignment: MTRKSDS
   - Access: Random
   - Record Key: MTR-CUST-ID
   - Purpose: Meter master file output
   - **Key Insight**: Random access for efficient meter record storage

4. **TO01-METER-ERR** (Lines 26-29)
   - Type: Sequential output file
   - Assignment: METERERR
   - Purpose: Error records output
   - **Key Insight**: Separate error file for data quality management

#### File Operations (Lines 149-200)
```cobol
2100-OPEN-FILES  SECTION.
    OPEN INPUT TI01-METER-FILE.
    IF NOT METER-IO-STATUS
       DISPLAY 'ERROR OPENING METER INPUT FILE'
       STOP RUN
    END-IF.

    OPEN INPUT MI01-CUSTOMER-KSDS.
    IF NOT CUST-KSDS-IO-STATUS
       DISPLAY 'ERROR OPENING CUSTOMER KSDS'
       STOP RUN
    END-IF.

    OPEN OUTPUT MO01-METER-KSDS.
    IF NOT KSDS-IO-STATUS
       DISPLAY 'ERROR OPENING METER KSDS'
       STOP RUN
    END-IF.

    OPEN OUTPUT TO01-METER-ERR.
    IF NOT ERR-IO-STATUS
       DISPLAY 'ERROR OPENING METER ERROR FILE'
       STOP RUN
    END-IF.
```
- **Purpose**: Open all four files with comprehensive error checking
- **Key Insight**: Always check file status after OPEN operations

#### Customer Processing Loop (Lines 147-160)
```cobol
2200-READ-CUSTOMER-KSDS  SECTION.
    READ MI01-CUSTOMER-KSDS
        AT END SET CUST-KSDS-EOF TO TRUE
        NOT AT END
            MOVE CUST-KEY TO WS-HARDCODED-CUST-ID
            PERFORM 2300-PROCESS-METER-DATA
    END-READ.
```
- **Pattern**: Standard file reading loop
- **Purpose**: Process each customer and their meter data

#### Meter Data Processing (Lines 160-200)
```cobol
2300-PROCESS-METER-DATA.
    READ TI01-METER-FILE
        AT END
            DISPLAY 'METER FILE EOF - SKIPPING CUSTOMER: ' WS-HARDCODED-CUST-ID
            ADD 1 TO WS-SKIP-CTR
        NOT AT END
            ADD 1 TO WS-READ-CTR
            PERFORM 2400-VALIDATE-METER-DATA
    END-READ.
```
- **Logic**: Read meter file for each customer
- **Error Handling**: Handle end of meter file gracefully

#### Meter Data Validation (Lines 200-240)
```cobol
2400-VALIDATE-METER-DATA.
    SET VALID-RECORD-FLAG TO TRUE.
    
    IF IN-PREV-READ IS NUMERIC AND IN-CURR-READ IS NUMERIC
        MOVE IN-PREV-READ TO WS-PREV-READING
        MOVE IN-CURR-READ TO WS-CURR-READING
        IF WS-CURR-READING >= WS-PREV-READING
            PERFORM 2500-GENERATE-METER-ID
            PERFORM 2600-WRITE-METER-KSDS
        ELSE
            DISPLAY 'INVALID READING - CURRENT < PREVIOUS FOR: ' WS-HARDCODED-CUST-ID
            SET ERROR-RECORD-FLAG TO TRUE
            PERFORM 2700-WRITE-ERROR-RECORD
        END-IF
    ELSE
        DISPLAY 'NON-NUMERIC READING DATA FOR: ' WS-HARDCODED-CUST-ID
        SET ERROR-RECORD-FLAG TO TRUE
        PERFORM 2700-WRITE-ERROR-RECORD
    END-IF.
```
- **Validation Rules**:
  1. Both readings must be numeric
  2. Current reading must be >= previous reading
- **Error Handling**: Invalid data sent to error file

#### Meter ID Generation (Lines 250-260)
```cobol
2500-GENERATE-METER-ID.
    MOVE WS-HARDCODED-CUST-ID(1:1) TO WS-MTR-CUST-CH1.
    MOVE WS-HARDCODED-CUST-ID(2:1) TO WS-MTR-CUST-CH2.
    MOVE WS-DD TO WS-MTR-DD.
    MOVE WS-MM TO WS-MTR-MM.
    COMPUTE WS-RAND-SEED = FUNCTION RANDOM * 10000.
    COMPUTE WS-MTR-RAND = FUNCTION MOD(WS-RAND-SEED, 10000).

    STRING WS-MTR-PREFIX WS-MTR-CUST-CH1 WS-MTR-CUST-CH2
           WS-MTR-DD WS-MTR-MM WS-MTR-RAND
           DELIMITED BY SIZE
           INTO MTR-ID
    END-STRING.
```
- **ID Format**: MTR- + Customer chars(1,2) + DDMM + 4-digit random
- **Example**: MTR-CJ12051234
- **Uniqueness**: Customer ID + Date + Random ensures uniqueness

#### Meter Record Writing (Lines 260-280)
```cobol
2600-WRITE-METER-KSDS.
    MOVE WS-HARDCODED-CUST-ID TO MTR-CUST-ID.
    MOVE WS-PREV-READING TO MTR-PREV-READ.
    MOVE WS-CURR-READING TO MTR-CURR-READ.

    WRITE MO01-METER-RECORD
        INVALID KEY
            DISPLAY 'DUPLICATE METER ID: ' MTR-ID
            ADD 1 TO WS-DUP-CTR
        NOT INVALID KEY
            ADD 1 TO WS-WRITE-CTR
    END-WRITE.
```
- **Purpose**: Write validated meter records to VSAM
- **Error Handling**: Handle duplicate meter ID errors

#### Error Record Writing (Lines 270-280)
```cobol
2700-WRITE-ERROR-RECORD.
    MOVE IN-PREV-READ TO ERR-PREV-READ.
    MOVE IN-CURR-READ TO ERR-CURR-READ.
    WRITE TO01-METER-ERR-RECORD.
    ADD 1 TO WS-ERROR-CTR.
```
- **Purpose**: Preserve invalid data for analysis
- **Data Integrity**: Original data maintained in error file

#### Termination Section (Lines 290-303)

- **Statistics**: Comprehensive processing report

## Key Programming Concepts

### 1. File Synchronization
- **Multiple Files**: Coordinate reading from customer and meter files
- **Sequential Processing**: Process customers in order
- **Error Recovery**: Handle missing or invalid meter data

### 2. Data Validation
- **Numeric Validation**: Ensure readings are valid numbers
- **Logical Validation**: Current reading must be >= previous
- **Business Rules**: Enforce meter reading constraints

### 3. ID Generation
- **Unique Identification**: Generate unique meter IDs
- **Random Components**: Use random numbers for uniqueness
- **Date Components**: Include date for temporal identification

### 4. Error Management
- **Separation**: Invalid data in separate error file
- **Preservation**: Maintain original data for analysis
- **Statistics**: Track all error types

## Viva Questions and Answers

### Q1: What is the purpose of the MID001 program?
**Answer**: "MID001 processes meter reading data by reading customer records, validating meter readings, generating unique meter IDs, and creating meter records in a VSAM file while handling errors appropriately."

### Q2: How are meter IDs generated?
**Answer**: "Meter IDs are generated using the format MTR- + first 2 characters of customer ID + day + month + 4-digit random number. For example: MTR-CJ12051234."

### Q3: What validation rules are applied to meter readings?
**Answer**: "The program validates that both previous and current readings are numeric, and that the current reading is greater than or equal to the previous reading to prevent backward meter readings."

### Q4: How does the program handle missing meter data?
**Answer**: "When the meter file reaches EOF before all customers are processed, the program displays a skip message, increments the skip counter, and continues with the next customer."

### Q5: What is the purpose of the error file?
**Answer**: "The error file preserves invalid meter reading data for later analysis. It maintains the original data format so that data quality issues can be investigated and corrected."

### Q6: How are duplicate meter IDs handled?
**Answer**: "When a duplicate meter ID is detected during WRITE operation, the program displays a duplicate message, increments the duplicate counter, and continues processing."

### Q7: Why is the customer KSDS accessed sequentially?
**Answer**: "The customer KSDS is accessed sequentially to process all customers in order, ensuring that each customer gets exactly one meter record from the corresponding position in the meter file."

### Q8: What is the difference between the meter input file and meter KSDS?
**Answer**: "The meter input file contains raw reading data (previous and current readings only), while the meter KSDS contains complete meter records with generated IDs and customer references."
